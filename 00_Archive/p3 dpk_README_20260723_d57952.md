# Automated Customer Reviews – NLP Project

An end‑to‑end NLP system that **classifies** customer reviews by sentiment, **clusters** products into business categories, and **summarises** large volumes of feedback into concise, actionable articles. The solution is deployed as a live web application with a React frontend and a FastAPI backend.

---

## 📌 Problem Statement

With thousands of reviews scattered across multiple platforms, manually extracting insights is slow, inconsistent, and unscalable. This project automates the process using modern NLP techniques, helping businesses and consumers quickly understand product sentiment, discover category trends, and get digestible summaries of customer feedback.[reference:0]

---

## 🎯 Project Goal

Develop a review‑aggregation system that:

- **Classifies** review sentiment (positive / neutral / negative)
- **Clusters** products into meaningful categories (e.g., *Tablets*, *E‑Readers*, *Smart Assistants*)
- **Summarises** reviews using generative AI to produce recommendation‑style articles[reference:1]

---

## 📊 Dataset

**Primary dataset:** [Consumer Reviews of Amazon Products](https://www.kaggle.com/datasets/datafiniti/consumer-reviews-of-amazon-products/data) (Datafiniti)

- ~34,000 reviews for products like Kindle, Fire TV Stick, and more
- Includes product info, ratings, review text, and other metadata
- Original data: `original_kaggle_data_download_archive.zip` (3 CSV files, merged during preprocessing)[reference:2]

---

## 🧩 Main Tasks

### 1. Sentiment Analysis
- Fine‑tuned **DistilBERT‑base‑uncased** for 3‑class sentiment classification
- Model quantized to ONNX and served via a FastAPI endpoint[reference:3][reference:4]

### 2. Product Category Clustering
- Hybrid approach: **TF‑IDF** vectorisation + **K‑Means** clustering (7 clusters) on cleaned product names
- Clusters interpreted and mapped to **7 business categories**:
  1. Tablets
  2. Tablets with Alexa
  3. Tablets for Kids
  4. E‑Readers
  5. Smart Assistants
  6. Chargers & Adapters
  7. Others[reference:5][reference:6]

### 3. Review Summarisation
- **Meta‑Llama‑3.2‑3B‑Instruct** generates concise, human‑readable summaries for product categories
- Summaries are generated on‑demand or cached for performance[reference:7][reference:8]

---

## ⚙️ Pipeline

1. **Text Preprocessing** – lowercasing, punctuation removal, stopword filtering, stemming/lemmatisation[reference:9]
2. **Feature Engineering** – TF‑IDF with n‑grams for clustering; tokenisation for sentiment and summarisation models[reference:10]
3. **Modelling** – DistilBERT (sentiment), K‑Means + TF‑IDF (clustering), Llama‑3.2 (summarisation)
4. **Deployment** – ONNX‑optimised models served via FastAPI on Vercel Serverless Functions; React frontend hosted on Vercel[reference:11][reference:12]

## 🔧 Setup & Installation

### Prerequisites
- Python 3.10+
- Node.js 16+ (for frontend)
- Hugging Face account (for model hosting)

### Backend (FastAPI)

```bash
# Clone the repository
git clone https://github.com/wtraquinas/project-3-brief-NLP-automated-customers-reviews.git
cd project-3-brief-NLP-automated-customers-reviews/05_Deployment/Review\ Sentiment\ Lab\ v2

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # or `venv\Scripts\activate` on Windows

# Install dependencies
pip install -r requirements.txt

# Set environment variables (optional, if using a custom model repo)
export HF_MODEL_REPO=AntonioTrx99/sentiment_model_quantized

# Run the API locally
uvicorn api.app:app --reload --port 8000
